# Документация по функционалу "Похожие лоты" и "Активные лоты с таким же кадастровым номером"

## 1. Общее описание
Функционал призван улучшить пользовательский опыт на страницах **архивных лотов** (торги по которым завершены, отменены и т.д.). Вместо того чтобы показывать пользователю тупиковую страницу с блоком "Как купить лот" (что неактуально для завершенных торгов), система предлагает альтернативы:
1. **Активные торги с тем же кадастровым номером** (если объект снова выставлен на торги).
2. **Похожие лоты** (по региону, категории и цене).

## 2. Архитектура и компоненты

### 2.1. Фоновый расчет похожих лотов (`SimilarLotsWorker.cs`)
Расположение: `b:\Projects\parser\Lots.Infrastructure\Services\SimilarLots\SimilarLotsWorker.cs`

Расчет похожих лотов — тяжелая операция, поэтому она выполняется в фоне отдельным воркером в проекте `FedresursScraper`.

**Логика работы:**
1. Воркер запускается раз в сутки в заданный час (настройка `RunAtHour`).
2. Берет пачками (настройка `BatchSize`) архивные лоты, для которых похожие лоты еще не считались или считались более 24 часов назад.
3. Для каждого архивного лота ищет до 4 **активных** похожих лотов.
4. **Алгоритм поиска (2 шага):**
   - **Шаг 1 (Strict):** Ищет активные лоты в том же регионе (`PropertyRegionCode`), с пересекающимися категориями и ценой в диапазоне ±50% от начальной цены исходного лота.
   - **Шаг 2 (CategoryFallback):** Если по жесткому фильтру найдено менее 4 лотов, добирает недостающие лоты по всей России только по совпадению категорий.
5. Результаты сохраняются в таблицу `SimilarLots` (сущность `SimilarLot`). Старые связи для этого лота предварительно удаляются.

**Настройки (appsettings.json):**
```json
"SimilarLots": {
  "Enabled": true,
  "RunAtHour": 2, // Час запуска (по UTC)
  "BatchSize": 100 // Размер пачки для обработки
}
```

### 2.2. API (`LotsController.cs`)
Расположение: `b:\Projects\parser\Lots.WebApi\Controllers\LotsController.cs`

Метод `GetLotAsync(string id)` отвечает за выдачу данных лота на фронтенд.

**Логика:**
Если запрашиваемый лот **неактивен** (`!lot.IsActive()`), контроллер делает два дополнительных запроса:
1. **Похожие лоты:** Читает из таблицы `SimilarLots` предрассчитанные лоты, сортирует по рангу (`Rank`) и мапит в `SimilarLotDto`.
2. **Лоты с тем же кадастровым номером:** Если у исходного лота есть `CadastralInfos`, берет список кадастровых номеров и ищет в БД **активные** лоты, у которых есть хотя бы один из этих номеров.
   - Найденные лоты помещаются в свойство `SameCadastralLots`.
   - Чтобы избежать дублирования, эти лоты исключаются из списка `SimilarLots`.

### 2.3. Фронтенд (`LotDetailsClient.tsx`)
Расположение: `b:\Projects\lot-app\app\lot\[lotId]\LotDetailsClient.tsx`

На фронтенде проверяется статус лота функцией `isFinalStatus(lot.tradeStatus)`.
Если статус финальный, скрывается блок "Как купить лот" и отображаются два новых блока:
1. **Блок "Активные лоты с этим кадастровым номером"** (если `lot.sameCadastralLots` не пуст). Имеет выделяющийся дизайн (светло-оранжевый фон, иконка предупреждения), чтобы сразу привлечь внимание пользователя к тому, что этот же объект сейчас торгуется.
2. **Блок "Похожие лоты"** (если `lot.similarLots` не пуст). Отображается в виде сетки карточек (фото, заголовок, цена).

Стили для этих блоков находятся в `lot.module.css` (`.similarLotsGrid`, `.similarLotCard`, `.sameCadastralTopBlock` и т.д.).

## 3. Как протестировать
1. Найти лот с финальным статусом (например, "Завершенные", "Торги отменены").
2. Найти активный лот.
3. Добавить им одинаковый кадастровый номер в БД.
4. Открыть страницу архивного лота — должен появиться оранжевый блок с активным лотом.
5. Для проверки "Похожих лотов" убедиться, что воркер `SimilarLotsWorker` отработал и заполнил таблицу `SimilarLots` для тестируемого архивного лота.
